# 2차시: NotebookLM 특징 및 장단점 소개 & 실습

## 🎯 학습 목표

1. NotebookLM의 핵심 개념과 일반 AI(ChatGPT, Claude 등)와의 차이점을 설명할 수 있다.
2. NotebookLM의 UI 구성과 주요 기능(소스 관리, 채팅, 스튜디오)을 이해하고 직접 사용할 수 있다.
3. 분양 공고문·제안서·리포트 등 실무 문서를 업로드하여 핵심 정보를 추출할 수 있다.
4. NotebookLM의 콘텐츠 제작 기능을 활용해 문서 초안·요약 자료를 생성할 수 있다.

## 📖 본 강의

### 1. 📓 NotebookLM이란?

> 구글이 만든 **AI 리서치 도우미** — 내가 올린 자료 안에서만 답변

![NotebookLM 3패널 UI](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/02_image/notebooklm_3panel_ui.gif)

NotebookLM은 Google이 개발한 **AI 기반 문서 분석 도구**
- 여러 문서를 한 곳에 모아두고, AI를 통해 질문·요약·인사이트 도출이 가능

🔗 **공식 사이트**: [notebooklm.google.com](https://notebooklm.google.com/)

---

### 💡 NotebookLM이 특별한 이유

ChatGPT, Claude 같은 일반 AI는 **학습된 전체 데이터**를 기반으로 답변
→ 가끔 없는 내용을 그럴듯하게 지어내는 **할루시네이션(환각)**이 발생

![할루시네이션 예시](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/02_image/hallucination_example.png)

반면 NotebookLM은 **업로드한 자료 범위 안에서만 답변**하고, **출처(인용)까지 표시**
→ "이 답변은 업로드한 문서의 몇 페이지에서 가져왔다"는 식으로 근거를 보여줌

| 구분 | 일반 AI (ChatGPT, Claude 등) | NotebookLM |
|------|------|------|
| **답변 근거** | 학습된 전체 데이터 | 내가 업로드한 자료만 |
| **할루시네이션** | 발생 가능 ⚠️ | 최소화 ✅ |
| **출처 표시** | 없거나 불확실 | 원문 위치까지 정확히 표시 |
| **최신 정보** | 웹 검색 가능 (유료) | 업로드한 자료에 한정 |
| **비용** | 무료 제한 / 유료 구독 | **무료** (구글 계정만 필요) |

> 💡 NotebookLM = "나만의 자료 전문가". 100페이지짜리 보고서를 올려놓고 "3페이지 내용 요약해줘"라고 하면, 정확히 3페이지 내용만 가져와서 답변

### 2. 🖥️ NotebookLM 화면 구성

![NotebookLM UI 구성](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/02_image/notebooklm_ui.png)

크게 **3개 패널**로 구성:

| 번호 | 영역 | 설명 |
|------|------|------|
| ❶ **출처 (소스 관리)** | 왼쪽 패널 | PDF·문서·웹자료 등 소스 업로드 영역. `+ 추가` 버튼으로 업로드 |
| ❷ **탐색** | 왼쪽 패널 상단 | 업로드한 소스를 검색·탐색하는 기능 |
| ❸ **채팅** | 가운데 패널 | 소스 범위 안에서 질문·답변. 답변에 **인용(출처)** 표시 |
| ❹ **스튜디오** | 오른쪽 패널 | 소스 기반 **콘텐츠 자동 생성**: AI 오디오 오버뷰, 동영상 개요, 마인드맵, 보고서, 플래시카드, 퀴즈 |
| ❺ **공유 / 설정** | 상단 | 노트북 공유 및 언어 설정 등 |

### 3. ✅ NotebookLM 장단점 정리

| 구분 | 내용 |
|------|------|
| ✅ **할루시네이션 최소화** | 내가 올린 자료 안에서만 답변 → AI가 지어낼 여지가 없음 |
| ✅ **출처 표시** | 답변 근거가 되는 원문 위치를 클릭 한번으로 확인 가능 |
| ✅ **무료** | 구글 계정만 있으면 무료 사용 가능 |
| ✅ **다양한 콘텐츠 생성** | 보고서, 마인드맵, 오디오 오버뷰, 퀴즈, 블로그 포스트 등 |
| ✅ **Gemini 연동** | NotebookLM 노트북을 Gemini에 직접 업로드하여 문서 작성 가능 |
| ✅ **팀 공유** | 노트북 공유로 팀 단위 공동 작업 지원 |
| ❌ **웹 검색 불가** | 업로드한 자료 외의 최신 정보는 활용 불가 |
| ❌ **Word/PPT 직접 업로드 불가** | 구글 문서/슬라이드로 변환 필요 |
| ❌ **소스 용량 제한** | 개별 소스 500,000 단어 제한 (매우 긴 문서는 분할 필요) |

> 💡 NotebookLM은 **완전 자동화 도구**라기보다는 **"연구 보조원"**에 가까움. 자료를 분석하고 쟁점을 도출하는 데 최적화

### 4. 🖥️ [시연] NotebookLM 기본 사용법

#### Step 1: 노트북 만들기
1. [notebooklm.google.com](https://notebooklm.google.com/) 접속 (구글 계정 로그인)
2. `+ 새 노트북` 클릭
3. 노트북 이름 설정 (예: "수도권 분양시장 분석")

#### Step 2: 소스 업로드하기
1. 왼쪽 패널 `+ 추가` 클릭
2. 업로드 방법 선택:
   - **파일 업로드**: PDF, 텍스트 파일 등 직접 업로드
   - **Google Drive**: 구글 드라이브에서 문서 가져오기
   - **웹사이트 URL**: 웹페이지 주소 붙여넣기
   - **텍스트 복사/붙여넣기**: 직접 텍스트 입력
3. 업로드 완료 후 소스 목록에서 확인

#### Step 3: 질문하기
1. 가운데 채팅 영역에 질문 입력
2. AI가 **업로드한 소스 기반**으로 답변 생성
3. 답변 내 **인용 번호 클릭** → 원문 해당 부분으로 바로 이동

### 5. 🖥️ [실습] 분양 공고문·제안서·리포트 핵심 정보 추출

> 📌 **실습 목표**: 분양 관련 실무 문서를 NotebookLM에 업로드하고, 핵심 정보를 빠르게 추출

#### 실습 준비
- NotebookLM 접속 → 새 노트북 생성
- 강사가 제공한 **분양 공고문 PDF** (또는 웹 URL)를 소스로 업로드

#### 실습 ① 분양 공고문 핵심 정보 추출

소스 업로드 후 아래 프롬프트 입력:

```
이 분양 공고문에서 다음 항목을 추출하여 표로 정리해줘:
1. 사업지 위치 및 규모 (총 세대수, 동수, 층수)
2. 분양 일정 (모집공고일, 청약접수일, 당첨자 발표일, 계약일)
3. 평형별 분양가격 범위
4. 입주 예정일
5. 시행사·시공사 정보
6. 특별공급 비율 및 조건 요약
```

> 💡 100페이지가 넘는 공고문도 **몇 초 만에** 핵심 정보만 추출 가능. 인용 번호 클릭 시 원문 해당 페이지로 바로 이동 → 검증도 쉬움

#### 실습 ② 제안서 분석

분양 마케팅 제안서를 추가 업로드한 뒤:

```
이 분양 마케팅 제안서에서 다음 내용을 분석해줘:
1. 제안사의 핵심 마케팅 전략 3가지를 요약
2. 타겟 고객군은 어떻게 정의하고 있는지
3. 미디어 믹스 및 예산 배분 계획
4. 경쟁 단지 대비 차별화 포인트
5. 이 제안서에서 가장 설득력 있는 부분과 보완이 필요한 부분
```

#### 실습 ③ 시장 리포트 요약

부동산 시장 리포트를 추가 업로드한 뒤:

```
이 부동산 시장 리포트에서 분양기획자 관점으로 다음을 정리해줘:
1. 전체 시장 동향 핵심 3줄 요약
2. 수도권 vs 지방 분양시장 비교 (표로 정리)
3. 분양가 추이 및 청약 경쟁률 변화
4. 향후 전망 및 주의해야 할 리스크 요인
5. 우리 단지 기획에 참고할 핵심 데이터 3개
```

> 💡 여러 문서를 동시에 올려놓으면 **교차 분석**도 가능. 공고문 3개를 올려놓고 "3개 단지의 분양가를 비교해줘"라고 하면 비교표를 자동 생성

### 6. 🖥️ [실습] NotebookLM 기반 문서 초안 / 자료 생성

> 📌 **실습 목표**: NotebookLM에 여러 소스를 올려놓고, 실무에서 바로 쓸 수 있는 문서 초안을 생성

#### 실습 ④ 경쟁 분석 보고서 초안 생성

여러 경쟁 단지 자료를 소스로 업로드한 뒤:

```
업로드한 경쟁 단지 자료들을 비교 분석하여
'경쟁 분석 보고서' 초안을 작성해줘. 다음 구성으로:

1. 개요: 분석 대상 단지 목록 및 분석 목적
2. 기본 정보 비교표 (위치, 세대수, 평형, 분양가, 입주시기)
3. 각 단지의 강점/약점 분석
4. 마케팅 전략 비교 (광고 채널, 타겟층, 핵심 메시지)
5. 우리 단지의 차별화 전략 제안
6. 결론 및 시사점
```

#### 실습 ⑤ 데이터 비교표 자동 생성

여러 단지 공고문·시장 보고서를 소스로 업로드한 뒤:

```
업로드한 자료에서 다음 데이터를 추출하여 비교표로 정리해줘:

| 단지명 | 위치 | 총 세대수 | 주력 평형 | 3.3㎡당 분양가 | 청약 경쟁률 | 입주시기 |

- 분양가는 만원 단위로 통일
- 경쟁률은 1순위 기준
- 데이터가 없는 항목은 "자료 없음"으로 표시
```

> 💡 NotebookLM은 **정확한 데이터 추출과 구조화**에 강점. 여러 문서에 흩어진 숫자를 한 번에 표로 정리 → 수작업으로 엑셀에 옮기는 시간을 대폭 절감

```
- 각 항목마다 근거가 되는 소스와 해당 페이지를 인용으로 표시해줘
```

> 💡 NotebookLM은 **모든 답변에 인용(출처)을 표시** → 보고서의 데이터 신뢰성을 바로 검증 가능. "이 숫자가 맞나?" 싶을 때 인용 번호 클릭 한 번이면 원문 확인 끝

#### 실습 ⑦ 스튜디오 — 인포그래픽 & 보고서 자동 생성

스튜디오 패널을 활용한 원클릭 콘텐츠 생성:

1. 오른쪽 **스튜디오** 패널에서 **"보고서"** 클릭 → 소스 기반 체계적 보고서 자동 생성
2. **"마인드맵"** 클릭 → 시장 분석 결과를 시각적 다이어그램으로 변환
3. 생성된 콘텐츠를 복사하여 PPT·문서에 바로 활용

> 💡 스튜디오의 보고서·마인드맵은 소스 데이터 기반으로 생성 → **수치 오류 없이** 신뢰할 수 있는 자료. 상사 보고 시 "AI가 만들었지만 데이터는 원본 그대로"라고 할 수 있음

### 8. 🔗 NotebookLM + Gemini 연계 활용 (미리보기)

> 다음 차시(3차시)에서 자세히 다룰 예정. 간단히 미리 소개

NotebookLM에서 분석한 노트북을 **Gemini에 직접 업로드**하여 더 정교한 문서 작성 가능

| 도구 | 역할 |
|------|------|
| **NotebookLM** | 자료 분석, 쟁점 도출, 핵심 정보 추출 |
| **Gemini** | 분석 결과 기반으로 긴 문서 초안 작성 (건의서, 보고서 등) |

```
[워크플로우]
① 분양 관련 자료를 NotebookLM에 업로드
② NotebookLM으로 핵심 정보 추출 & 분석
③ NotebookLM 노트북을 Gemini에 소스로 연결
④ Gemini에서 최종 보고서·제안서 초안 작성
```

> 💡 하나의 AI에 다 맡기는 게 아니라, **각 도구의 강점을 살려서 단계별로 조합**하는 것이 핵심

## 📝 요약 정리

### 1. NotebookLM = "나만의 자료 전문가"
- 내가 올린 자료 안에서만 답변 → **할루시네이션 최소화**
- 답변마다 **출처(인용) 표시** → 원문 검증 쉬움
- **무료** + 구글 계정만 있으면 OK

### 2. NotebookLM 핵심 기능 3가지
- **소스 관리**: PDF, 웹URL, 텍스트 등 다양한 자료 업로드
- **채팅**: 소스 기반 질의응답 (인용 표시)
- **스튜디오**: 보고서, 마인드맵, 오디오 오버뷰 등 콘텐츠 자동 생성

### 3. 분양기획 실무 활용법
- 공고문·제안서·리포트 **핵심 정보 빠르게 추출**
- 여러 문서에서 **데이터 추출 → 비교표 자동 생성**
- 시장 요인 분석 보고서 / **SWOT 분석** 초안 생성 (인용 출처 포함)
- 스튜디오 활용 **인포그래픽·마인드맵·보고서** 원클릭 생성
