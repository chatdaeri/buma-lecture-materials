# 1. 📌 노코드 자동화란? (15분)

## 노코드 자동화의 정의

> 💡 **노코드 자동화** = 코딩 없이, 시각적 인터페이스로 여러 앱·서비스 간의 데이터 흐름과 작업을 자동화하는 방법

쉽게 말해, **여러 서비스를 클릭만으로 연결해서** 자동으로 일하게 만드는 것

![기존 코딩 방식 vs Make](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/07_image/code_vs_make.png)

- 왼쪽(기존 방식): 자동화하려면 개발자가 직접 코드를 작성해야 했음
- 오른쪽(Make): **클릭만으로** 구글 폼과 구글시트를 바로 연결할 수 있음

## 자동화 가능한 업무 예시 (분양 기획 관련)

| 분류 | 자동화 예시 |
|------|------------|
| 📧 고객 응대 | 상담 신청 접수 → 자동 확인 메일/카톡 발송 |
| 📊 데이터 수집 | 네이버 뉴스에서 특정 키워드(경쟁 단지명, 분양가 등) 기사 자동 수집 |
| 📱 콘텐츠 발행 | AI로 생성한 홍보 문구 → 인스타·블로그 자동 포스팅 |
| 📋 보고서 작성 | 구글시트 데이터 → AI 요약 → 보고서 자동 생성 |
| 📅 일정 관리 | 상담 예약 → 캘린더 자동 등록 + 담당자 알림 |
| 🔔 모니터링 | 경쟁사 SNS/뉴스 변동 감지 → 슬랙/이메일 알림 |

## ⚠️ 자동화에 적합한 업무 vs 부적합한 업무

| | 자동화에 **적합한** 업무 ✅ | 자동화에 **부적합한** 업무 ❌ |
|---|---|---|
| **특징** | 반복적이고 프로세스가 고정된 작업 | 실험적이고 매번 달라지는 작업 |
| | 투자 대비 효과가 명확함 | 투자 대비 효과가 불명확함 |
| **예시** | 매일 반복되는 분양현황 보고 자동화 | 매번 스타일이 바뀌는 분양 홍보 영상 기획 |
| | 고객별 맞춤 확인 메일 자동 발송 | 아직 테스트 중인 신규 마케팅 캠페인 |
| | 경쟁 단지 뉴스 모니터링 | 창의적 판단이 필요한 기획서 작성 |

> 💡 자동화의 핵심: **"매일 하는 일"**, **"순서가 정해진 일"**, **"시간을 아끼면 돈이 되는 일"**부터 시작하는 것이 좋음

# 2. 📌 바이브 코딩 vs 노코드 자동화 (10분)

## 바이브 코딩이란?

최근 **클로드 코드, 커서(Cursor)** 같은 AI 코딩 도구가 등장하면서 비개발자도 자연어로 명령하면 코드를 자동 작성할 수 있게 됨. 이를 **바이브 코딩(Vibe Coding)**이라 부름

그렇다면 "AI한테 코드 짜달라고 하면 되는데, 굳이 노코드 툴을 배워야 하나?" 라는 의문이 생길 수 있음

## 바이브 코딩 vs 노코드 자동화 비교

|  | 🖥️ 바이브 코딩 | ⚙️ 노코드 자동화 (Make 등) |
|---|---|---|
| **화면 예시** | ![바이브 코딩 화면](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/07_image/vibe_coding_example.png) | ![Make 시나리오 화면](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/07_image/make_scenario_example.png) |
| **작동 방식** | AI에게 자연어로 명령 → 코드 자동 생성 | 미리 만들어진 앱 블록을 시각적으로 연결 |
| **장점** | 복잡한 코드도 자연어로 작성 가능 | 워크플로우 흐름을 한눈에 파악 가능, 새로 코드를 짤 필요 없이 기존 기능 조합 |
| **단점** | 문제 발생 시 어디가 잘못됐는지 파악 어려움, 매번 바닥부터 새로 생성해야 함 | 기본 기능을 배워야 함, 입력 화면(UI) 구현은 별도 필요 |
| **비유** | 🏌️ **바람 부는 날 골프** — 방향은 내가 정하지만, AI가 코드를 어떻게 짜느냐에 따라 결과가 달라질 수 있음 | 🎮 **골프 시뮬레이터** — 원하는 대로 공을 보낼 수 있지만, 게임 조작법을 배워야 함 |

## 🤔 그래서 뭘 써야 할까?

| 상황 | 추천 도구 |
|------|----------|
| 여러 앱을 연결해서 **반복 업무를 자동화** 하고 싶을 때 | ✅ **노코드 자동화** (Make 등) |
| 나만의 **웹사이트/앱/크롤러** 등을 만들고 싶을 때 | ✅ **바이브 코딩** (Cursor, Claude Code 등) |
| 대량 데이터를 **수집·분석·가공** 하고 싶을 때 | ✅ **바이브 코딩** 또는 **둘 다 조합** |
| 팀원 누구나 **유지보수** 할 수 있어야 할 때 | ✅ **노코드 자동화** (시각적이라 인수인계 쉬움) |

> 💡 업무 자동화 관점에서는 **노코드 자동화가 더 유리**할 수 있음. 기존에 있는 앱과 서비스를 엮어서 처리하는 업무가 대부분이기 때문. Make를 먼저 배우고, 바이브 코딩까지 익히면 자동화 범위를 더 넓힐 수 있음

# 3. 📌 노코드 자동화 툴 3종 비교 (15분)

## 대표적인 노코드 자동화 툴

노코드 자동화 시장에서 가장 많이 쓰이는 3대 도구를 비교해봄

### Zapier (제피어) — 가장 쉬운 입문용

![Zapier 로고](https://upload.wikimedia.org/wikipedia/commons/thumb/9/98/Zapier_logo.png/480px-Zapier_logo.png)

- [zapier.com](https://zapier.com)
- 2012년 출시, **노코드 자동화의 원조**
- "원하는 자동화"를 문장으로 입력하면 AI가 워크플로우 자동 생성
- **가장 쉬운** 진입장벽, 입문자에게 추천

### Make (메이크) — 가성비 + 기능의 균형

![Make 연동 앱 목록](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/07_image/make_integrations.png)

- [make.com](https://make.com)
- 구 Integromat, **3,000개 이상** 앱 연동
- 시각적 캔버스 방식으로 **복잡한 흐름도 한눈에** 파악 가능
- 기능, 가격, 난이도 면에서 **가장 균형** 잡힌 도구

### n8n (엔에잇엔) — 개발자 친화적, 무료 사용 가능

- [n8n.io](https://n8n.io)
- 커뮤니티(오픈소스) 버전 **완전 무료** (단, 자체 서버에 설치해야 함)
- JavaScript 코드로 **복잡한 커스텀 로직** 구현 가능
- MCP 연동, AI 에이전트 등 **고급 기능**이 편리함

## 📊 3종 비교표

|  | 제피어 (Zapier) | 메이크 (Make) | n8n (엔에잇엔) |
|---|---|---|---|
| **학습 난이도** | ⭐ 쉬움 | ⭐⭐ 중간 | ⭐⭐⭐ 어려움 |
| **월 무료 한도** | 100 Tasks, 2단계 자동화만 가능 | **1,000 크레딧**, 고급 기능(웹훅 등) 포함 | 커뮤니티 버전 **무료 무제한** (자체 설치 필요) |
| **유료 요금** | $29.99/월~ | **$9/월~** | €20/월~ (클라우드) |
| **연동 앱 수** | 8,000개+ | 3,000개+ | 약 500개 |
| **시각화 방식** | 선형 (위→아래) | **캔버스** (자유 배치) | 노드 (자유 배치) |
| **주요 특징** | AI가 워크플로우 자동 생성, 입문자에게 편리 | 워크플로우가 시각적, **확장성 뛰어남** | JS 코드 가능, MCP·AI 에이전트 편리, **가격 효율적** |
| **한계** | 무료는 2단계만, **가격이 가장 비쌈** | 기능·가격·난이도 균형 (특별한 한계 없음) | 무료 사용엔 서버 설치 등 **개발 지식 필요** |
| **추천 대상** | 자동화 입문자 | **비개발자 중급 사용자** | 개발 경험이 있는 사용자 |

## 💡 왜 이 강의에서는 Make를 사용하는가?

| 이유 | 설명 |
|------|------|
| **1. 가성비** | 무료 1,000크레딧으로 충분한 실습 가능. 유료도 $9/월로 가장 저렴 |
| **2. 시각적** | 캔버스 방식으로 복잡한 자동화 흐름을 한눈에 볼 수 있음 |
| **3. 확장성** | 기본기를 익히면 n8n 같은 고급 도구로 넘어가기도 수월함 |
| **4. 무료 범위** | Zapier와 달리, 무료에서도 웹훅·HTTP 등 고급 기능 사용 가능 |

> 💡 Make 하나만 제대로 익혀두면, 지루하고 반복적인 업무를 자동화하여 **시간과 비용을 획기적으로 절약**할 수 있음

## ⚙️ [핵심 개념 01] 시나리오와 모듈

Make에서 가장 중요한 2가지 개념:

### 시나리오 (Scenario) = 자동화 워크플로우

- 하나의 **자동화 플로우 전체**를 시나리오라고 부름
- 예: "상담 신청 → 시트 기록 → 확인 메일 발송" 이 전체가 1개의 시나리오

### 모듈 (Module) = 시나리오를 구성하는 각 단계

- 시나리오 안의 **각각의 앱/동작 블록**을 모듈이라 부름
- 예: 위 시나리오에서 "Gmail - Send Email"이 1개의 모듈

![모듈 개념도 (Gmail→Telegram)](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/07_image/make_module_concept.png)

위 그림은 2개의 모듈로 구성된 시나리오 예시:
- **Gmail** 모듈: 이메일 가져오기 (1 오퍼레이션)
- **Telegram Bot** 모듈: 텍스트 메시지 발송 (1 오퍼레이션)
- 이 시나리오가 1번 실행되면 → 총 **2 오퍼레이션 = 2 크레딧** 소모

### 4가지 모듈 유형

| 유형 | 아이콘 | 설명 | 예시 |
|------|--------|------|------|
| **Trigger** | ⏰ 시계 | 시나리오의 **시작점** (항상 첫 번째) | 폼 제출 감지, 이메일 수신 감지 |
| **Action** | ⚡ 번개 | 데이터 **생성/수정/삭제** 실행 | 이메일 발송, 시트에 행 추가 |
| **Search** | 🔍 돋보기 | 데이터 **조회/검색** | 구글시트에서 특정 행 찾기 |
| **Universal** | 🔧 렌치 | HTTP 요청 등 **범용** 기능 | 외부 API 호출 |

## ⚙️ [핵심 개념 02] 커넥션 (Connection)

커넥션이란, **Make와 외부 서비스 간의 연결**을 설정하는 것

![커넥션 설정 화면](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/07_image/make_connection.png)

- Gmail 모듈이 "내 메일함에서 메일을 가져오려면" → **내 Gmail 계정과 Make 사이에 커넥션**을 먼저 연결해야 함
- 최초 1회만 설정하면 이후에는 자동으로 연결됨
- 앱을 추가할 때 'Create a connection' 팝업이 뜨므로, 안내를 따라 쉽게 연결 가능

## ⚙️ [핵심 개념 03] 오퍼레이션과 크레딧

| 용어 | 의미 |
|------|------|
| **오퍼레이션 (Operation)** | 모듈이 **1번 동작**한 것. 모듈 1개 = 1 오퍼레이션 |
| **크레딧 (Credit)** | 과금 단위. 기본적으로 **1 오퍼레이션 = 1 크레딧** 차감 |

### 요금제 비교

| 요금제 | 무료 (Free) | 코어 (Core) | 프로 (Pro) |
|--------|-------------|-------------|------------|
| 월 요금 | **$0** | $10.59 | $18.82 |
| 크레딧 | 1,000회/월 | 10,000회/월 | 10,000회/월 |
| 활성 시나리오 | 2개 | 무제한 | 무제한 |
| 주요 기능 | 2,000개+ 앱, 웹훅/HTTP 포함 | 무료 기능 + 시나리오 무제한 | 코어 + 커스텀 변수, 우선 실행 |

### 💰 무료 1,000크레딧으로 얼마나 쓸 수 있나?

5개 모듈로 구성된 시나리오 기준:
- 1번 실행 = 5 크레딧 소모
- 무료 1,000크레딧 ÷ 5 = **약 200번** 실행 가능

> 💡 무료 요금제만으로도 충분히 자동화를 실습하고 실무에 적용할 수 있음. 처음에는 부담 없이 무료로 시작하는 것을 추천
