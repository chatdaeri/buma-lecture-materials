# 1. 📌 도입 — 왜 리서치용 AI가 필요한가? (5분)

## 분양 기획자의 일상 속 리서치

분양 기획자는 하루에도 수십 건의 정보를 찾아야 함

```
"이 지역 분양가 상한제 적용 여부가 어떻게 되지?"
"경쟁 단지 분양률이 얼마나 되는지 확인해야 하는데..."
"건분법 시정명령 관련 최신 판례가 있나?"
"분양 홍보 영상에서 핵심 내용만 빠르게 뽑고 싶은데..."
```

이런 리서치 업무에 **기존 방식**(네이버·구글 검색 → 링크 10개씩 클릭 → 내용 읽고 정리)은 시간이 너무 오래 걸림

### ⚡ AI 리서치 도구를 쓰면?

| 기존 방식 | AI 리서치 도구 |
|-----------|---------------|
| 검색 → 링크 클릭 → 읽기 → 정리 | 질문 → 구조화된 답변 + 출처 |
| 1건 리서치에 30분~1시간 | 1건 리서치에 **3~5분** |
| 출처 확인 어려움 | **출처 링크 자동 제공** |
| 여러 자료 교차 비교 어려움 | **AI가 교차 분석** 후 정리 |

> 💡 리서치용 AI는 "검색"을 넘어 **"분석·정리·검증"까지** 가능. 단, 도구마다 강점이 다르기 때문에 **목적에 맞는 도구 선택**이 중요

---

## ⚠️ 그런데, AI 검색을 그냥 믿어도 될까? — 할루시네이션 문제

AI 리서치 도구는 강력하지만, **치명적인 약점**이 존재

> 🚨 **할루시네이션(Hallucination)** = AI의 거짓말
> AI는 모르는 걸 "모르겠습니다"라고 하지 않고, **그럴듯하게 지어냄.**
> 존재하지 않는 판례를 인용하거나, 없는 통계 수치를 만들어내기도 함.

![할루시네이션 개념 설명](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/01_image/hallucination.png)

분양 기획 업무에서 이게 왜 위험할까?
- 없는 판례 번호를 근거로 보고서를 작성 시 → **신뢰도 추락**
- 잘못된 분양가 통계로 기획안을 만들 경우 → **의사결정 오류**

### 🔬 실제 실험으로 확인된 AI 할루시네이션 — Ahrefs 가짜 브랜드 실험

SEO 분석 도구 기업 Ahrefs가 진행한 실험:
- **완전히 가짜인 럭셔리 문진(페이퍼웨이트) 브랜드 "Xarumei"를 만들고**,
- 공식 웹사이트, Medium 블로그, Reddit 등에 서로 모순되는 정보를 퍼뜨린 뒤,
- **8개 AI 플랫폼에 56개 질문**을 던짐

> 📌 존재하지 않는 브랜드에 대해 "유명인이 이 브랜드를 홍보했다는데?", "결함 제품 논란은 어떻게 됐어?" 같은 **가짜 전제가 깔린 질문**을 던진 셈

### 📊 AI별 반응 결과

| AI 플랫폼 | 반응 | 설명 |
|-----------|------|------|
| **ChatGPT** | ✅ 가장 우수 | 56개 질문 중 53~54개 정답, 거짓 정보 합성을 거부 |
| **Claude** | ✅ 매우 우수 | "존재하지 않는 브랜드"라고 일관되게 판단 (100% 회의적) |
| **Perplexity** | ⚠️ 약 40% 오답 | 가짜 브랜드를 Xiaomi와 혼동하기도 함 |
| **Grok** | ❌ 취약 | 거짓 창립자, 도시, 생산량을 사실로 반복 |
| **Copilot** | ❌ 취약 | 세 가지 거짓 출처의 정보를 혼합하여 제시 |
| **Gemini** | ⚠️ 불안정 | 초기 "존재하지 않음" → 나중에 거짓 정보 채택으로 변경 |

### 💡 이 실험이 알려주는 것

1. **AI는 "가장 상세하고 구체적인 답변"을 선호** → 거짓이라도 구조화된 정보가 있으면 채택할 수 있음
2. **AI마다 할루시네이션 수준이 다름** → 하나의 AI만 믿지 말고, **여러 도구를 교차 검증**해야 함
3. **출처 확인은 필수** → AI가 제시한 판례 번호, 통계, 인용구는 반드시 원문 확인

> 💡 **그래서 오늘 배울 핵심:**
> 여러 리서치 AI 도구의 특징을 알고, **서로 교차 검증하면서 사용하는 방법**

# 2. 🔍 리서치용 AI 도구 소개 (15분)

## 2-1. Perplexity — AI 검색의 대표 주자

🔗 [perplexity.ai](https://www.perplexity.ai/)

**Perplexity**는 자연어로 질문하면 여러 출처를 **교차 분석**해서 구조화된 답변을 주는 AI 검색 엔진

### ✅ 핵심 특징
- 구글 검색처럼 링크 10개를 나열하는 게 아니라, **AI가 직접 읽고 정리**해서 보고서 형태로 답변
- 모든 답변에 **출처 번호** 표시 → 원문 확인 바로 가능
- **딥 리서치(Deep Research)** 기능: 복잡한 주제를 수십 개 출처에서 동시에 검색·교차 분석

### 📌 언제 쓰면 좋을까?
- 시장 동향, 경쟁사 분석, 뉴스 기반 리서치
- 법령·판례 등 출처가 중요한 조사
- "이 주제에 대해 전반적으로 알려줘" 같은 **탐색형 리서치**

---

## 2-2. ChatGPT 웹 검색 — 가장 익숙한 AI의 검색 기능

🔗 [chat.openai.com](https://chat.openai.com/)

ChatGPT는 대화형 AI의 대표 주자로, **웹 검색(Browse)** 기능을 통해 실시간 정보를 가져올 수 있음

### ✅ 핵심 특징
- 대화 맥락을 이해한 상태에서 **추가 검색** 가능 (예: "방금 결과에서 서울만 따로 정리해줘")
- GPTs, 이미지 생성 등 다른 기능과 **조합 활용** 가능
- **Deep Research** 기능: 최대 30분간 종합적인 조사를 수행 (Plus/Pro 구독자)

### 📌 언제 쓰면 좋을까?
- 검색 결과를 바탕으로 **후속 질문을 이어가며** 점점 깊이 파고들 때
- 검색 + 분석 + 문서 작성을 **하나의 대화 안에서** 해결하고 싶을 때

---

## 2-3. Claude 웹 검색 — 장문 분석에 강한 AI의 검색

🔗 [claude.ai](https://claude.ai/)

Anthropic의 Claude도 **웹 검색** 기능을 지원

### ✅ 핵심 특징
- **긴 문서 처리에 강점** — 검색 결과를 가져와 긴 보고서나 분석문을 작성하는 데 탁월
- **정확하고 신중한 답변 스타일** — "모르겠다"고 솔직하게 말하는 경향 (할루시네이션 상대적으로 적음)
- 출처와 함께 **구조화된 분석** 제공 가능

### 📌 언제 쓰면 좋을까?
- 검색 결과를 바탕으로 **긴 보고서·제안서 초안**을 작성할 때
- 복잡한 정책·법률 내용을 **분석·비교**할 때

---

## 2-4. Google AI Search (AI Overview) — 검색의 본가

🔗 구글 검색창에서 바로 사용 가능

구글 검색 결과 상단에 나타나는 **AI Overview** 기능

### ✅ 핵심 특징
- 별도 가입 없이 **구글 검색만 하면** AI 요약 자동 표시
- 구글의 방대한 검색 인덱스 기반 → **가장 넓은 범위의 웹 데이터** 활용
- 검색 결과 내 관련 링크 함께 제시

### 📌 언제 쓰면 좋을까?
- 빠르게 개요를 파악하고 싶을 때
- 기존 구글 검색 습관 그대로 AI 요약을 받고 싶을 때

## 2-5. NotebookLM — 내 자료 안에서만 답변하는 AI

🔗 [notebooklm.google.com](https://notebooklm.google.com/)

Google이 만든 AI 리서치 도구로, **내가 올린 자료 범위 안에서만 답변**

### ✅ 핵심 특징

| 구성 | 설명 |
|------|------|
| **소스 관리** | PDF·문서·웹자료 등을 업로드하여 지식 파일로 사용 |
| **프롬프트 입력창** | 업로드한 소스 범위 안에서 질문에 답하고, **인용(근거)으로 원문 위치를 함께 제공** |
| **콘텐츠 제작** | 보고서, 인포그래픽, 슬라이드 자료 등 소스 기반 콘텐츠 생성 |
| **메모 기능** | 분석 결과를 메모로 저장하여 정리 |
| **공유** | 노트북 공유로 팀 단위 공동 작업 지원 |

> 📎 **업로드 가능 형식**: 구글 문서, 슬라이드, PDF, 텍스트(.txt), 마크다운(.md), 웹 URL, 오디오 파일
> ⚠️ **업로드 불가**: Word(.docx), PowerPoint(.pptx) → 구글 문서/슬라이드로 변환 후 업로드

### 🛡️ 할루시네이션이 걱정될 때 → NotebookLM

앞서 본 Ahrefs 실험처럼 일반 AI는 **없는 내용을 지어낼 수 있음**. 하지만 NotebookLM은 업로드한 자료 안에서만 답변하고 **출처까지 표시**해주기 때문에, 정확성이 중요한 업무에 특히 유용

### 📌 언제 쓰면 좋을까?
- 공고문, 제안서, 리포트 등 **특정 문서의 핵심 정보를 추출**할 때
- 여러 문서를 올려놓고 **교차 분석·쟁점 도출**할 때
- 할루시네이션 없이 **정확한 답변이 필요**할 때

> 💡 NotebookLM은 2차시에서 더 자세히 다루면서 직접 실습 예정

## 2-6. Genspark — 멀티에이전트 교차검증 검색

🔗 [genspark.ai](https://www.genspark.ai/)

Genspark은 **여러 AI 모델을 동시에 활용**해서 답변을 주는 멀티에이전트 검색 엔진

### ✅ 핵심 특징
- **슈퍼 에이전트**: ChatGPT, Claude, Gemini 등 여러 AI를 동시에 활용 → 교차검증으로 **신뢰도 향상**
- **팩트 체크 에이전트**: 텍스트나 링크를 넣으면 웹상의 신뢰할 수 있는 소스와 대조하여 **사실 여부를 검증**
- **AI 시트**: "XX 목록 조사해줘" → 스프레드시트 자동 완성
- **AI 슬라이드**: 자료를 업로드하면 AI가 요약·분석해서 슬라이드로 제작

### 📌 언제 쓰면 좋을까?
- AI가 생성한 정보의 **사실 여부를 검증**하고 싶을 때
- 여러 AI의 답변을 **한 번에 비교**하고 싶을 때
- 리서치 결과를 **표나 슬라이드로 바로 정리**하고 싶을 때

# 3. 🖥️ [시연] 동일 프롬프트 4개 도구 비교 + Deep Research (10분)

## 3-1. 같은 질문, 다른 답변 — 4개 도구 비교

> 💡 **시연 목적**: 같은 프롬프트를 넣었을 때 각 도구가 어떤 스타일로 답변하는지 직접 비교

### 📝 시연용 프롬프트

```
2025-2026년 수도권 신규 분양 아파트 시장 전망과 분양률 추이를 분석해줘.
특히 다음 내용을 포함해줘:
1. 수도권 주요 지역별(서울, 경기, 인천) 분양 물량 현황
2. 최근 분양률(청약 경쟁률) 추이와 변화 요인
3. 분양가 상한제 적용 확대에 따른 시장 영향
4. 분양 기획자가 주목해야 할 트렌드 3가지
```

### 🖥️ [시연] 비교 포인트

| 비교 항목 | Perplexity | ChatGPT | Claude | Google AI Search |
|-----------|------------|---------|--------|-----------------|
| **답변 형식** | 구조화된 보고서 + 출처 번호 | 대화체 분석 | 체계적 장문 분석 | 간결한 요약 |
| **출처 표시** | ✅ 번호+링크 | ✅ 링크 | ✅ 링크 | ✅ 관련 링크 |
| **후속 질문** | 가능 | 매우 강점 | 가능 | 제한적 |
| **데이터 정확도** | 높음 (다중 소스) | 중간 | 높음 (신중한 답변) | 중간 |
| **답변 깊이** | 깊음 | 깊음 | 매우 깊음 | 보통 |

> 🎓 **수강생 체크포인트**: 각 도구의 답변을 보면서 "내 업무에는 어떤 스타일이 가장 유용할까?" 생각해볼 것

## 3-2. Deep Research 기능 비교 — 더 깊이 파고들기

"일반 검색"과 **"Deep Research"**의 차이:

> 일반 검색 = 질문 → **즉시 답변** (몇 초)
> Deep Research = 질문 → AI가 **스스로 검색 계획을 세우고, 수십 개 소스를 탐색·분석** → 종합 보고서 (수 분~30분)

### 📊 Deep Research 3대 도구 비교

| 구분 | ChatGPT Deep Research | Gemini Deep Research | Genspark Deep Research |
|------|----------------------|---------------------|----------------------|
| **구독 요건** | Plus ($20/월) 또는 Pro | Advanced ($19.99/월) | 무료 플랜 있음 |
| **소요 시간** | 최대 **30분** (가장 깊이있는 분석) | **수 분** 내 | **수 분** 내 |
| **특징** | 단계별 추론 과정 공개, 가장 상세 | Google 검색 인덱스 연동, 한국어 자료 풍부 | 멀티에이전트 교차검증 |
| **결과물** | 긴 리포트 형태 | 구조화된 보고서 | 검증된 종합 답변 |
| **추천 상황** | 깊이 있는 시장 분석, 경쟁사 조사 | 한국어 자료 중심 리서치 | 빠른 교차검증이 필요할 때 |

### 🖥️ [시연] Deep Research 비교 시연

```
"수도권 분양시장 2025-2026년 심층 분석:
분양가 상한제 확대, 금리 변동, 인구 이동 패턴이
분양 기획에 미치는 영향을 종합적으로 분석해줘"
```

> 같은 프롬프트를 ChatGPT Deep Research / Gemini Deep Research / Genspark에 넣고 결과를 비교

**비교 시 주목할 점:**
- 📊 **데이터의 최신성**: 어떤 도구가 가장 최신 통계를 가져오는가
- 📑 **출처의 신뢰도**: 공식 통계(국토부, 한국부동산원) vs 뉴스 기사 vs 블로그
- 📝 **분석의 깊이**: 단순 나열 vs 인과관계 분석 vs 시나리오 제시

# 4. 🖥️ [시연] Perplexity → Genspark 팩트체크 워크플로우 (3분)

> 💡 **핵심 원칙**: AI가 검색한 자료도 100% 신뢰하면 안 됨. **AI가 생성한 정보를 다른 AI로 교차 검증**하는 습관 필요

## 워크플로우 개요

| 단계 | 도구 | 역할 |
|------|------|------|
| ① 자료 수집 | **Perplexity** | 분양 관련 판례·법령·기사 등 딥 리서치로 수집 |
| ② 팩트 체크 | **Genspark** | 수집된 정보의 사실 여부를 교차 검증 |

## 🖥️ [시연] Step 1 — Perplexity로 자료 수집

```
2024년부터 2026년 현재까지 건축물의 분양에 관한 법률(건분법) 위반에 따른
분양계약 해제 관련 대법원 및 하급심 판례를 조사해줘.
```

→ Perplexity가 여러 출처를 교차 검색해서 **판례 번호, 판결 요지, 관련 행정해석**까지 구조화해서 보여줌

> ⚠️ 판례 번호나 법조항은 **AI가 지어낼 수 있는 영역**. 반드시 검증 필요

## 🖥️ [시연] Step 2 — Genspark으로 팩트 체크

### 사용 방법
1. [genspark.ai](https://www.genspark.ai/) 접속 → 로그인
2. 화면 중간 우측 **'모든 에이전트'** 클릭
3. **'팩트 체크'** 에이전트 선택
4. Perplexity에서 수집한 내용 중 검증이 필요한 부분을 복사·붙여넣기

### 팩트 체크 프롬프트

```
첨부된 내용들의 사실 여부를 웹상의 신뢰할 수 있는 소스와 대조하여 검증해주세요.
```

→ Genspark이 각 항목별로 **"✅ 확인됨", "⚠️ 부분적으로 정확", "❌ 확인 불가"** 같은 식으로 판정

> 💡 **실무 팁**: 특히 **판례 번호, 법조항 번호, 통계 수치** 같은 팩트 기반 정보는 이 과정을 반드시 거칠 것
