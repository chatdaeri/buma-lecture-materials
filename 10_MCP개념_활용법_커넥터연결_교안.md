# 1. 📌 도입 — MCP, 왜 주목받고 있는가? (5분)

## AI + 외부 도구 = 업무 자동화

지금까지 배운 AI 도구들(ChatGPT, Claude, Gemini 등)은 **대화는 잘하지만**, 혼자서는 이런 일을 못 함:

```
"노션에서 지난주 회의록 찾아와줘"
"구글 캘린더에 내일 오전 10시 미팅 잡아줘"
"네이버에서 '수원역 분양' 키워드 상위 블로그 분석해줘"
"경쟁 단지 홈페이지 데이터 긁어와줘"
```

AI가 아무리 똑똑해도 **외부 도구에 접근하는 방법**이 없으면 대화 안에서 끝나버림

## ⚡ 그래서 등장한 것이 MCP

사실, **AI + 외부 도구** 조합이 강력하다는 건 이미 증명된 사실:

| 서비스 | 조합 | 결과 |
|--------|------|------|
| **Perplexity** | AI + 검색 기능 | 실시간 리서치 AI |
| **Cursor** | AI + 개발 도구 | AI 코딩 에디터 |
| **NotebookLM** | AI + 문서 분석 | 문서 기반 AI 조수 |

이처럼 AI에 외부 도구를 붙이면 엄청난 시너지가 나지만, **MCP 이전에는** 이런 연동을 하려면 처음부터 모든 것을 직접 개발해야 했음

> 💡 MCP는 이 문제를 해결하기 위해 **Anthropic(클로드 개발사)이 2024년에 발표한 표준 규칙(프로토콜)**. AI와 외부 도구를 연결하는 방법을 **규격화**한 것

MCP가 나온 이후, 구글·OpenAI 등 모든 주요 AI 기업이 MCP를 지원하기 시작하면서 **사실상 AI 업계의 표준**이 됨

# 2. 🔧 MCP 개념 — AI에게 도구를 쥐어주는 방법 (12분)

## 2-1. MCP란?

**MCP (Model Context Protocol)** = AI가 외부 도구에 접근하고 사용할 때, 그 방법을 **표준화한 규칙(프로토콜)**

마치 인터넷에서 **HTTP**라는 통신 규칙을 쓰는 것과 같이, AI와 외부 도구가 통신할 때의 방법을 규칙화한 것

![MCP 프로토콜 아키텍처 개념도](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/10_image/mcp_architecture.png)

> 💡 쉽게 비유하면: **USB 포트**와 같은 역할
> - USB 이전: 프린터, 키보드, 마우스마다 **각각 다른 단자**가 필요했음
> - USB 이후: **하나의 규격**으로 뭐든 꽂으면 됨
> - MCP도 마찬가지: AI에 노션, 캘린더, 크롤러 등을 **하나의 방식**으로 연결

## 2-2. MCP 작동 방식

```
[AI 클라이언트]          [MCP 서버]           [외부 서비스]
Claude Desktop   ←→   MCP 서버    ←→   노션
Claude.ai        ←→   MCP 서버    ←→   구글 캘린더
Cursor           ←→   MCP 서버    ←→   네이버 검색
                                  ←→   웹 크롤러(Firecrawl, Apify)
```

- **AI 클라이언트**: Claude, ChatGPT 같은 AI 앱 (사용자가 대화하는 곳)
- **MCP 서버**: AI와 외부 서비스를 연결해주는 **중간 다리** 역할
- **외부 서비스**: 노션, 구글 캘린더, 네이버, 웹사이트 등 실제 도구

## 2-3. MCP vs 기존 방식

![MCP 이전 vs MCP 이후](https://raw.githubusercontent.com/chatdaeri/buma-lecture-materials/master/10_image/mcp_before_after.png)

| 구분 | 기존 방식 (API 직접 구현) | MCP 방식 |
|------|-------------------------|---------|
| **통합 방법** | 도구마다 별도 개발 필요 | 표준 프로토콜로 통일 |
| **난이도** | 개발자만 가능 | 비개발자도 설정 가능 |
| **유연성** | 하나 바꾸면 전체 수정 | 서버만 교체하면 됨 |
| **재사용성** | 낮음 | 높음 (누구나 만든 MCP 서버 공유 가능) |

## 2-4. MCP의 장단점

### ✅ 장점
- AI 채팅창 안에서 필요한 데이터를 **바로 가져올 수 있음** → 중간중간 외부 자료 참고가 쉬움
- **자연어로 명령** 가능 → 복잡한 단계가 간단해짐
- 한번 연결해두면 **반복 활용** 가능

### ⚠️ 단점
- 기존 API 방식에 비해 **속도가 다소 느림**
- 가끔 명령 수행에 **실패**할 수 있음
- **보안상 우려** — 외부 서비스 접근 권한을 AI에게 주는 것이므로 주의 필요

# 3. 💼 MCP 활용사례 — 이런 일이 가능해짐 (5분)

## 실무 활용사례 4가지

### 사례 1: 📅 팀 회의 자동 조율
- **연결**: Google Calendar
- **명령**: "팀원들 캘린더 확인해서, 다들 가능한 시간으로 회의 잡아줘"
- **효과**: 수동으로 시간 맞추던 작업을 AI가 자동 처리

### 사례 2: 📂 노션 자료 → 보고서 자동 작성
- **연결**: Notion
- **명령**: "노션에서 OOO분양 관련 정보 찾아와서, 블로그 글 작성해줘"
- **효과**: 노션에 흩어진 자료를 AI가 취합하여 초안 생성

### 사례 3: 🔍 경쟁사 분석 자동화
- **연결**: 네이버 서치 MCP + 구글 드라이브
- **명령**: "경쟁사OO의 웹, 뉴스, 검색량 등을 조사해 보고서로 만들어서 구글드라이브에 저장해줘"
- **효과**: 검색 → 분석 → 보고서 작성 → 저장까지 한 번에

### 사례 4: 🕷️ 웹 데이터 대량 수집
- **연결**: Apify MCP
- **명령**: "네이버 부동산에서 '마포래미안푸르지오' 매물 스크랩해서 표로 정리해줘"
- **효과**: 수작업 크롤링 없이 AI가 자동 수집·정리

# 4. 🔄 커넥터 vs 로컬 MCP — 뭐가 다른가? (5분)

클로드에서 외부 도구를 연결하는 방법은 크게 **2가지**:

| 구분 | 커넥터 (Connector) | 로컬 MCP 서버 |
|------|-------------------|--------------|
| **설치** | 클릭 몇 번으로 연결 ✅ | config 파일 편집 + 설치 필요 |
| **난이도** | 매우 쉬움 | 약간의 기술적 이해 필요 |
| **제공 방식** | 클로드가 공식 지원하는 앱만 가능 | 누구나 만든 MCP 서버 연결 가능 |
| **사용 환경** | claude.ai (웹) | Claude Desktop, Cursor 등 |
| **요금** | 클로드 Pro 구독 필요 ($20/월) | MCP 서버 자체는 무료 (AI 사용료 별도) |
| **대표 예시** | Notion, Google Calendar, Slack, Figma | Firecrawl, Naver Search, Apify 등 |

> 💡 **추천 전략**: 커넥터에 원하는 앱이 있으면 **커넥터 먼저 사용** (훨씬 편함). 커넥터에 없는 도구는 **로컬 MCP로 설치**

### 현재 클로드 커넥터에서 지원하는 주요 앱

| 앱 | 용도 |
|----|------|
| 📒 **Notion** | 문서·DB 검색, 페이지 생성·수정 |
| 📅 **Google Calendar** | 일정 조회·생성·수정 |
| 💬 **Slack** | 메시지 전송·채널 관리 |
| 🎨 **Figma** | 디자인 파일 접근·수정 |
| 🎨 **Canva** | 디자인 생성 |
| 📋 **Jira** | 프로젝트 관리·이슈 트래킹 |
| 📊 **HubSpot** | CRM·마케팅 자동화 |

# 7. 🖥️ [실습] Apify 커넥터 연결 & 사용해보기 (10분)

## Apify란?

**Apify** = 웹 크롤링·스크래핑을 쉽게 할 수 있게 해주는 클라우드 플랫폼

| 구분 | 설명 |
|------|------|
| **핵심 기능** | 웹사이트에서 데이터를 자동으로 수집 |
| **특징** | 1,600개 이상의 사전 제작된 크롤러(Actor) 제공 |
| **장점** | 코딩 없이 인스타그램, 네이버, 구글맵 등 대량 데이터 수집 가능 |
| **가격** | 월 $49부터 (무료 체험 가능) |

🔗 **공식 사이트**: [apify.com](https://apify.com)

### Firecrawl과 뭐가 다른가?

| 구분 | Firecrawl | Apify |
|------|-----------|-------|
| **용도** | 단일 웹페이지 스크래핑 | 대량 데이터 수집·크롤링 |
| **강점** | 빠르고 간편한 단건 처리 | SNS, 지도, 쇼핑 등 **특화 크롤러** 다수 |
| **사용 예** | 블로그 1개 내용 가져오기 | 인스타 해시태그별 게시물 100개 수집 |
| **MCP 지원** | ✅ | ✅ |

> 💡 한 줄 요약: **Firecrawl = 소량 정밀 스크래핑**, **Apify = 대량 자동 수집**

## Step 1: Apify MCP 연결

Apify는 현재 클로드 내장 커넥터에는 없으므로, **MCP 서버로 설치**해야 함

**① Apify 가입 & API 토큰 발급**
- [apify.com](https://apify.com) 가입
- Settings → Integrations → **API Token** 복사

**② Claude Desktop MCP 설정**

Claude Desktop 설정 파일에 아래 내용 추가:

```json
{
  "mcpServers": {
    "apify": {
      "command": "npx",
      "args": ["-y", "@apify/actors-mcp-server"],
      "env": {
        "APIFY_TOKEN": "your-apify-token-here"
      }
    }
  }
}
```

> 📁 **설정 파일 위치**
> - Mac: `~/Library/Application Support/Claude/claude_desktop_config.json`
> - Windows: `%APPDATA%\Claude\claude_desktop_config.json`

**③ Claude Desktop 재시작**
- 설정 저장 후 Claude Desktop을 완전히 종료했다가 다시 실행
- 채팅창에서 MCP 아이콘(🔌)이 보이면 연결 성공

## Step 2: Apify MCP 활용해보기

### 🖥️ [시연] 인스타그램 데이터 수집

```
프롬프트 예시:
"Apify를 사용해서 인스타그램에서 '#래미안' 해시태그로
최근 게시물 20개를 수집하고, 좋아요 수 기준으로 정리해줘"
```

### 🖥️ [시연] 부동산 관련 데이터 수집

```
프롬프트 예시:
"Apify를 사용해서 구글맵에서 '강남구 부동산 중개업소'를 검색하고,
상위 20개 업체의 이름, 주소, 평점, 리뷰 수를 표로 정리해줘"
```

### 🖥️ [시연] 네이버 블로그 대량 수집

```
프롬프트 예시:
"Apify로 네이버 블로그에서 '마포 래미안 분양' 키워드의
최근 30일간 게시물을 수집해서 제목, 날짜, URL을 정리해줘"
```

> 💡 **9차시에서 배운 Make.com과의 차이**: Make.com도 데이터 수집이 가능하지만, **SNS·네이버 등 대량 데이터 수집은 Apify가 훨씬 효율적**. Make.com은 수집 후 **후처리 자동화**(메일 발송, 시트 저장 등)에 강점

# 8. 📝 정리 & 다음 차시 예고 (3분)

## ✅ 오늘 배운 핵심

| 항목 | 내용 |
|------|------|
| **MCP란** | AI가 외부 도구에 접근하는 **표준 프로토콜** (USB 포트와 같은 역할) |
| **커넥터** | 클로드 웹에서 **클릭만으로** 연결 가능한 공식 지원 앱 (노션, 캘린더 등) |
| **로컬 MCP** | 설정 파일 편집으로 **더 다양한 도구** 연결 가능 (Apify, Firecrawl 등) |
| **활용 전략** | 커넥터에 있으면 커넥터 사용 → 없으면 로컬 MCP 설치 |

## 🔮 다음 차시 예고

> - 구글 캘린더 일정 → Gmail로 자동 전송
> - 프롬프트 → 캔바 디자인 만들기
> - 여러 MCP를 조합한 복합 자동화 실습
