# 1. 📌 도입 — Build 기능, 왜 배워야 하나? (3분)

## 4~5차시에서 배운 것

4차시에서 Google AI Studio의 **Playground, Build, Gallery** 3가지 인터페이스를 살펴봄. 5차시에서는 **나노바나나**로 이미지를 생성하고 편집하는 법을 배움

이번 시간에는 **Build 모드**를 본격적으로 활용하여 **실무에서 바로 쓸 수 있는 앱**을 직접 만들어 봄

```
"경쟁 단지 분양가를 한눈에 비교하는 차트가 필요한데..."
"분양 광고를 외국인 투자자용으로 빠르게 번역해야 하는데..."
"고객에게 인테리어 시뮬레이션을 바로 보여주고 싶은데..."
```

→ 이 모든 것을 **코딩 없이, Build 모드 하나로** 해결 가능

### ⚡ Build 모드 = "바이브 코딩(Vibe Coding)"

| 기존 방식 | Build 모드 |
|-----------|-----------|
| 개발자에게 의뢰 → 수주일 소요 | **자연어로 설명 → 수분 내 앱 완성** |
| HTML/CSS/JavaScript 지식 필요 | **코딩 지식 불필요** |
| 수정 시 재의뢰 | **대화로 바로 수정** |
| 별도 서버 필요 | **Google Cloud Run 원클릭 배포** |

> 💡 Build 모드는 설명만 하면 **React + Tailwind 기반의 완전한 웹앱**을 자동 생성함. "바이브 코딩"이란 **분위기(vibe)만 전달하면 코딩이 완성**된다는 뜻

🔗 **접속**: [aistudio.google.com](https://aistudio.google.com) → 왼쪽 사이드바 **Build** 클릭

# 2. 🏗️ Build로 도표 & 다이어그램 만들기 (12분)

## 2-1. Build 모드 시작하기

Google AI Studio에 접속 → 왼쪽 사이드바에서 **Build** 클릭

![Google AI Studio 메인 화면](04_image/google_ai_studio_ui.jpeg)

### Build 모드 시작 화면 3가지 옵션

| 옵션 | 설명 | 언제 사용? |
|------|------|-----------|
| ✏️ **프롬프트 입력** | 원하는 앱을 자연어로 설명 | 구체적인 아이디어가 있을 때 |
| 🎲 **I'm Feeling Lucky** | AI가 창의적 아이디어를 자동 제안 | 영감이 필요할 때 |
| 📚 **갤러리에서 리믹스** | 기존 앱을 복제해서 수정 | 참고할 만한 앱이 있을 때 |

### Build 모드가 생성하는 것

```
프롬프트 입력
    ↓
AI가 코드 자동 생성 (React + Tailwind CSS)
    ↓
즉시 실행 가능한 웹앱 미리보기
    ↓
수정 요청 → 대화로 반복 개선
    ↓
완성 후 배포 (Google Cloud Run) 또는 ZIP 다운로드
```

> 💡 생성된 코드를 이해할 필요 없음. **"이거 색상 바꿔줘", "버튼 추가해줘"** 같은 자연어 수정만 하면 됨

## 2-2. 🖥️ [시연] SVG 다이어그램 생성기 앱 만들기

PPT에 삽입할 **깔끔한 다이어그램과 도표**를 생성하는 앱을 Build로 만들어 봄

### Step 1: Build 모드에서 프롬프트 입력

```
분양 기획 업무용 다이어그램 생성기 앱을 만들어줘.

기능:
1. 사용자가 텍스트로 프로세스를 입력하면 SVG 플로우차트로 변환
2. 다이어그램 유형 선택 가능: 플로우차트, 조직도, 타임라인, 비교표
3. 색상 테마 선택 (비즈니스 블루, 모던 그레이, 브랜드 컬러 커스텀)
4. SVG 파일로 다운로드 가능 (PPT에 바로 삽입 가능)
5. 한국어 지원

디자인: 깔끔하고 전문적인 UI, 미리보기 영역은 넓게
```

### Step 2: 생성 결과 확인 & 수정

AI가 앱을 생성하면 우측에 **실시간 미리보기**가 표시됨

수정 요청 예시:
```
- "플로우차트 화살표를 둥근 모서리로 바꿔줘"
- "다운로드 버튼을 SVG와 PNG 두 가지로 만들어줘"
- "색상 팔레트에 '분양 브랜드' 테마 추가해줘 (메인: #2B5797, 포인트: #E74C3C)"
```

### Step 3: 실제 활용 — 분양 프로세스 다이어그램

앱에 다음 텍스트를 입력:

```
분양 마케팅 프로세스:
1단계: 시장 조사 & 입지 분석
2단계: 타깃 고객 설정
3단계: 분양가 책정 & 상품 기획
4단계: 홍보 전략 수립 (온/오프라인)
5단계: 모델하우스 오픈 & 현장 마케팅
6단계: 계약 & 사후 관리
```

→ **깔끔한 SVG 플로우차트가 자동 생성** → PPT에 바로 삽입 가능

## 2-3. 🖥️ [시연] PPT용 비교 도표 만들기

### Gemini에 직접 SVG 요청하는 방법 (Build 없이도 가능)

Google AI Studio **Playground**에서도 SVG를 직접 생성할 수 있음

```
프롬프트:
아래 데이터로 깔끔한 SVG 비교 차트를 만들어줘.
PPT에 삽입할 거니까 배경은 투명, 폰트는 14px, 전체 크기 800x400.

| 단지명 | 전용 84㎡ 분양가 | 프리미엄 |
|--------|-----------------|---------|
| A단지 | 5.2억 | +3,000만 |
| B단지 | 4.8억 | +1,500만 |
| C단지 | 5.5억 | +4,200만 |

스타일: 모던, 그라데이션 바 차트, 색상은 블루/네이비 계열
```

→ AI가 **SVG 코드**를 생성 → 복사해서 `.svg` 파일로 저장 → PPT에 삽입

### 💡 SVG를 PPT에 삽입하는 방법

1. AI가 생성한 SVG 코드를 복사
2. 메모장에 붙여넣기 → `비교차트.svg`로 저장
3. PPT에서 **삽입 → 그림 → 파일에서** 선택
4. 벡터 이미지이므로 **확대해도 깨지지 않음** ✨

| 이미지 형식 | 확대 시 | PPT 적합도 |
|------------|---------|-----------|
| PNG/JPG | 깨짐 (픽셀) | ⭐⭐ |
| **SVG** | **깨지지 않음 (벡터)** | ⭐⭐⭐⭐⭐ |

> 🎓 **수강생 체크포인트**: SVG는 벡터 형식이라 아무리 확대해도 선명함. PPT 보고서에 들어가는 도표는 SVG로 만드는 것이 프로페셔널함

# 3. 🌍 Ad Localizer — 분양 광고 다국어 현지화 (10분)

## 3-1. Ad Localizer란?

Google이 나노바나나 2(Gemini 3.1 Flash Image)로 시연한 **광고 현지화 데모 앱**

```
원본 한국어 분양 광고
        ↓
    Ad Localizer
        ↓
영어 / 중국어 / 일본어 / 베트남어 버전 자동 생성
(텍스트 번역 + 비주얼 현지화 동시 처리)
```

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 🔤 **인이미지 텍스트 변환** | 이미지 속 텍스트를 다른 언어로 자동 번역 |
| 🎨 **비주얼 현지화** | 번역된 텍스트에 맞게 레이아웃·디자인 자동 조정 |
| 🌐 **다국어 동시 생성** | 한 번에 여러 언어 버전 생성 가능 |
| 📐 **레이아웃 유지** | 원본 디자인의 구도와 스타일을 유지하면서 변환 |

🔗 **참고**: [Build with Nano Banana 2 - Google Blog](https://blog.google/innovation-and-ai/technology/developers-tools/build-with-nano-banana-2/)

## 3-2. 🖥️ [시연] 분양 광고 현지화 실습

### 분양 기획에서 Ad Localizer가 필요한 이유

- 외국인 투자자 대상 분양 마케팅 증가 (중국, 일본, 베트남, 미국 등)
- 기존: 디자이너에게 **언어별 별도 시안 제작 의뢰** → 시간·비용 소요
- Ad Localizer: **원본 광고 하나로 다국어 버전 자동 생성** → 즉시 활용

### Step 1: Google AI Studio Build에서 프롬프트 입력

```
분양 광고 다국어 현지화 앱을 만들어줘.

기능:
1. 사용자가 한국어 분양 광고 이미지를 업로드
2. 원하는 언어를 선택 (영어, 중국어 간체/번체, 일본어, 베트남어)
3. 나노바나나 API를 사용하여 이미지 내 텍스트를 번역하고 새 이미지 생성
4. 원본과 변환된 이미지를 나란히 비교
5. 변환된 이미지 다운로드

디자인: 깔끔하고 직관적, 드래그앤드롭 이미지 업로드
```

### Step 2: 결과 확인

```
[원본 - 한국어]                    [변환 - 영어]
┌─────────────────┐          ┌─────────────────┐
│  ★★ 래미안 ★★   │          │  ★★ Raemian ★★  │
│                 │   →→→    │                 │
│ "도심 속 자연,   │          │ "Nature in the  │
│  새로운 시작"    │          │  City, A New    │
│                 │          │  Beginning"     │
│ 분양가 5.2억~   │          │ From ₩520M~     │
│ 모델하우스 오픈  │          │ Model House Open│
└─────────────────┘          └─────────────────┘
```

> 💡 기존에는 디자이너가 별도로 작업하던 것을 **AI가 이미지 내 텍스트까지 자동 번역**함. 레이아웃과 디자인 톤은 그대로 유지됨

### 실무 활용 시나리오

| 상황 | 기존 방식 | Ad Localizer |
|------|----------|-------------|
| 외국인 투자자 IR 자료 | 번역 의뢰 3~5일 | **즉시 생성** |
| 해외 부동산 박람회 홍보물 | 언어별 시안 제작 | **한 번에 5개 언어** |
| SNS 다국어 광고 소재 | 언어별 별도 디자인 | **원본 하나로 자동** |

# 4. 🏠 나노바나나 + Build = 인테리어 디자인 앱 만들기 (15분)

## 4-1. 컨셉: 모델하우스 인테리어 시뮬레이터

5차시에서 나노바나나로 이미지를 편집하는 법을 배움. 이번에는 Build 모드와 결합하여 **누구나 사용할 수 있는 인테리어 시뮬레이션 앱**을 만들어 봄

```
고객이 빈 방 사진 업로드
        ↓
스타일 선택 (모던, 북유럽, 클래식, 한옥)
        ↓
나노바나나가 인테리어 적용된 이미지 생성
        ↓
고객이 직접 비교하며 스타일 결정
```

### 분양 기획에서의 가치

| 활용 장면 | 효과 |
|----------|------|
| 모델하우스 방문 전 | 고객이 온라인에서 **미리 인테리어 시뮬레이션** |
| 분양 상담 현장 | 고객 취향에 맞게 **즉시 시각화** 제공 |
| 옵션 상담 | 다양한 옵션(마감재, 가구 배치)을 **즉석 비교** |
| 마케팅 콘텐츠 | SNS·브로셔용 **다양한 인테리어 이미지 대량 생성** |

---

## 4-2. 🖥️ [시연] 인테리어 디자인 앱 만들기

### Step 1: Build 모드에서 프롬프트 입력

```
인테리어 디자인 시뮬레이터 앱을 만들어줘.

기능:
1. 사용자가 빈 방 또는 현재 인테리어 사진을 업로드
2. 인테리어 스타일 선택: 모던 미니멀, 북유럽, 클래식, 내추럴, 인더스트리얼
3. 추가 옵션: 가구 배치, 벽지 색상, 바닥재, 조명 분위기
4. 나노바나나(Gemini 이미지 생성)를 사용하여 인테리어 적용된 이미지 생성
5. Before/After 비교 슬라이더
6. 생성된 이미지 다운로드

디자인: 고급스러운 부동산 브랜드 느낌, 다크 모드 지원
```

### Step 2: AI가 앱 생성 → 미리보기 확인

Build 모드가 자동으로 생성하는 것:
- **이미지 업로드** 영역 (드래그앤드롭)
- **스타일 선택** 버튼/드롭다운
- **옵션 패널** (가구, 벽지, 바닥재 등)
- **생성 버튼** → 나노바나나 API 호출
- **Before/After 비교** 슬라이더
- **다운로드** 버튼

### Step 3: 수정 & 개선 요청

```
수정 요청 예시:
- "스타일 미리보기 이미지를 각 버튼 옆에 작게 보여줘"
- "생성 중에 로딩 애니메이션 추가해줘"
- "생성된 이미지 아래에 '이 스타일로 상담 신청' 버튼 추가해줘"
- "최근 생성 이력을 갤러리로 보여줘"
```

### Step 4: 실제 테스트

앱이 생성되면 직접 테스트:

1. 빈 방 사진 업로드 (또는 도면 이미지)
2. "북유럽 스타일" 선택
3. 옵션: "밝은 원목 바닥, 화이트 벽, 간접 조명"
4. **생성** 클릭 → 나노바나나가 인테리어 적용 이미지 생성
5. Before/After 슬라이더로 비교

> 💡 5차시에서 배운 나노바나나의 **선택적 편집** 기능이 여기서 활용됨. "벽지만 바꿔줘", "가구만 추가해줘" 같은 세부 조정도 가능

---

## 4-3. 배포하기 — 고객이 직접 사용하게 하려면?

Build에서 만든 앱은 **실제로 배포**할 수 있음

| 배포 방법 | 설명 | 비용 |
|----------|------|------|
| 🔗 **공유 링크** | Build 내 Share 버튼으로 즉시 공유 | 무료 |
| 🚀 **Google Cloud Run** | 정식 웹사이트로 배포 | 사용량 기반 과금 |
| 📦 **ZIP 다운로드** | 코드를 받아서 자체 서버에 배포 | 서버 비용 별도 |
| 🐙 **GitHub 연동** | 코드를 GitHub에 푸시 → 추후 수정 가능 | 무료 |

> 🎓 **수강생 체크포인트**: 분양 상담 현장에서 태블릿으로 이 앱을 열어 고객에게 바로 인테리어 시뮬레이션을 보여줄 수 있음. **공유 링크만 있으면 별도 설치 없이 웹브라우저에서 실행**됨

# 5. 💡 활용 아이디어 소개 (10분)

## 5-1. 맞춤형 브로셔 제작 앱

### 컨셉: 구글시트 + Gemini + 나노바나나 = 자동 브로셔

```
구글시트에 고객 정보 입력
(가구 구성, 선호 스타일, 투자 목적, 선호 평형)
        ↓
Gemini가 고객 맞춤형 문구 생성
        ↓
나노바나나가 맞춤형 이미지 생성
        ↓
자동으로 맞춤형 디지털 브로셔 완성
```

### Build 프롬프트 예시

```
맞춤형 분양 브로셔 생성기를 만들어줘.

입력:
- 고객 이름
- 가구 구성 (1인, 신혼부부, 3인가족, 4인가족+)
- 선호 스타일 (모던, 클래식, 내추럴)
- 투자 목적 (실거주, 투자, 학군)
- 선호 평형 (59㎡, 84㎡, 105㎡)

출력:
- 고객 맞춤형 문구가 포함된 디지털 브로셔
- 선호 스타일에 맞는 인테리어 이미지
- 해당 평형의 장점을 강조한 카피
- PDF 다운로드 가능
```

### 활용 시나리오

| 상황 | 효과 |
|------|------|
| 방문 상담 후 | 고객 정보 입력 → **10초 만에 맞춤형 브로셔** 생성 후 이메일 발송 |
| 온라인 사전 접수 | 폼 제출 시 자동으로 맞춤형 자료 발송 |
| 재방문 고객 | 이전 선호도 기반 **업데이트된 브로셔** 제공 |

> 💡 똑같은 브로셔를 모든 고객에게 나눠주는 것이 아니라, **고객별 관심사에 맞춘 브로셔**가 자동 생성됨. 고객 입장에서 "나를 위해 준비한 자료"라는 인상을 줌

## 5-2. AI 보이스 도슨트 — 모델하우스 음성 가이드

### 컨셉: STT + Gemini + TTS = 음성 대화형 안내 시스템

```
방문객이 모델하우스에서 질문 (음성)
        ↓
    STT (음성 → 텍스트)
        ↓
Gemini가 단지 정보 기반으로 전문 답변 생성
        ↓
    TTS (텍스트 → 음성)
        ↓
자연스러운 음성으로 안내
```

### 활용 기술

| 기술 | 역할 | Google API |
|------|------|-----------|
| **STT** | 방문객 음성 질문 인식 | Google Speech-to-Text (125+ 언어) |
| **Gemini** | 단지 정보 기반 전문 답변 생성 | Gemini API |
| **TTS** | 자연스러운 음성으로 안내 | Gemini 2.5 Flash/Pro TTS (99+ 언어) |

### Gemini TTS의 특징

| 기능 | 설명 |
|------|------|
| 🎭 **감정 표현** | 밝고 친절한 톤으로 안내 가능 |
| 🎯 **속도 조절** | 설명은 천천히, 숫자는 또박또박 |
| 👥 **멀티스피커** | 최대 2명 (남/여 가이드 번갈아) |
| 🌐 **다국어** | 한국어, 영어, 중국어, 일본어 등 자동 전환 |

🔗 **Gemini TTS 공식 문서**: [ai.google.dev/gemini-api/docs/speech-generation](https://ai.google.dev/gemini-api/docs/speech-generation)

### 실무 적용 예시

```
방문객: "이 단지 주변에 학교가 있나요?"

AI 도슨트: "네, 반경 1km 이내에 OO초등학교와 OO중학교가 있습니다.
도보 약 10분 거리이며, 통학 셔틀도 운행 예정입니다.
교육환경에 대해 더 자세한 정보가 필요하시면 말씀해주세요."
```

### Build로 프로토타입 만들기

```
모델하우스 AI 도슨트 프로토타입을 만들어줘.

기능:
1. 마이크 버튼을 눌러 음성 질문 입력 (STT)
2. 미리 입력된 단지 정보(면적, 가격, 학군, 교통 등)를 기반으로 Gemini가 답변 생성
3. 음성으로 답변 재생 (TTS)
4. 대화 내역을 텍스트로도 표시
5. FAQ 바로가기 버튼 (분양가, 입주일, 학군, 교통, 옵션)

디자인: 태블릿 최적화, 고급스러운 부동산 브랜드 느낌
```

> 💡 이 프로토타입은 **분양 마케팅 기획 제안서**에 "이런 시스템을 구축할 수 있다"는 데모 용도로 활용 가능. 실제 구축은 개발팀과 협업이 필요하지만, **기획자가 직접 프로토타입을 만들어 보여줄 수 있다는 것**이 핵심
